Personal Portfolio Website

Description:
A showcase of my journey into web development. This project was my first foray into using HTML and CSS. It's structured to highlight my projects, skills, and experiences, demonstrating the foundational practices of web page layout and design.

My Design Process

For my initial portfolio page, I aimed for a clean and minimalist design to show that a simple design can be effective for a webpage. I selected a color palette that reflects this aesthetic, using neutral tones with a few accent colors to create a visually appealing yet simple design:

#70BEBC (Soft Seafoam)

#F8F4DE (Warm Ivory)

#E22728 (Vibrant Coral)

I used Merriweather and Cardo as fonts to maintain readability and a clean look. One of the challenges I faced was struggling with Bootstrap initially, but I overcame this by practicing and gradually getting comfortable with its components and grid system.

Wireframe and Design Mockup

I used Figma to create the wireframe and design mockup for this project. Initially, I planned for a single-page design as seen in my wireframe. However, I realized that the page was becoming too long and cluttered, which led me to add an extra page in my design mockup to improve usability and navigation.

Wireframe


Design Mockup

My Coding Process

Technologies

For this first project, I focused on improving my HTML/CSS skills and also learned how to use basic Bootstrap to structure the layout of my page. This project was all about making clean HTML and using semantic code. I learned Bootstrap and how to utilize it for layout with flexboxes, and used CSS transitions and animations. I wanted the design to be simple, not overpowering, and easy to read.

Usability Testing

To ensure that my portfolio is user-friendly and accessible to a wide audience, I conducted usability tests with six participants from diverse backgrounds and age groups. The feedback from these tests helped identify areas for improvement and refine the overall design.

Participants

  • Engineering Major (22 years old)
  • Architect (25 years old)
  • Software Developer (36 years old)
  • Retired Doctor (68 years old)
  • Anthropology Major (18 years old)
  • Nurse (55 years old)

Testing Methodology

Each participant was asked to complete the following tasks:

  • Find and navigate to a specific project in the portfolio.
  • Describe the design process for one project by locating the provided details.
  • Provide feedback on the overall design, readability, and ease of use.

Event Countdown Timer

Description:
The Event Countdown Timer is an interactive web application designed to count down to any specified event. This project was developed using React.js to handle the stateful components efficiently and Bootstrap for responsive design, ensuring that the countdown timer is accessible on any device.

My Design Process

I aimed for a user-friendly interface with a minimalist design, focusing on ease of use and clarity. The design allows users to easily set and view the countdown to their events without unnecessary distractions.

Wireframe

Color Palette

The chosen color palette is designed to be visually soothing yet attention-grabbing, enhancing the user's experience while awaiting their event:

#F5F5F5 (Ghost White)

#F4F4F9 (Lavender Grey)

#3498DB (Dodger Blue)

#E74C3C (Carmine Red)

My Coding Process

Technologies

This project was an excellent opportunity to enhance my React.js skills, focusing on using state and props to manage the countdown timer dynamically. Bootstrap was utilized to ensure the layout is responsive and functional on various devices, from smartphones to desktops.

Usability Testing

To ensure the countdown timer was intuitive and met user needs, I conducted usability tests involving participants from different demographics. Feedback was crucial in refining the interface to make it as user-friendly as possible.

Participants

  • Engineering Major (22 years old)
  • Architect (25 years old)
  • Software Developer (36 years old)
  • Retired Doctor (68 years old)
  • Anthropology Major (18 years old)
  • Nurse (55 years old)

Participants were asked to set a countdown, pause, and reset the timer. Their ability to interact with the timer without guidance was monitored, and adjustments were made based on their experiences to enhance usability.

Interactive Quiz App

Description:
The Interactive Quiz App is a dynamic web application designed to offer users a fun and engaging way to test their knowledge across various topics. Developed using React.js for efficient state management and Bootstrap for responsive design, this app ensures an interactive and accessible experience on any device.

My Design Process

The goal was to create a user-friendly interface that is both engaging and intuitive, minimizing distractions to focus users on quiz content. The layout is designed to guide users through quizzes smoothly without any unnecessary complexity.

Wireframe

Color Palette

The selected color palette is designed to enhance focus and user engagement, using vibrant colors to differentiate between quiz elements and feedback:

#4a76a8 (Steel Blue): Used for primary interface elements and headers.

#f8f9fa (Cultured): Applied as the background color to provide a clean and subtle canvas.

#007bff (Vivid Blue): Utilized for interactive elements such as buttons and links to draw attention.

#28a745 (Forest Green): Used for correct answers and confirmation messages to signify success.

#dc3545 (Ruby Red): Employed for warnings and incorrect answers to alert users.

My Coding Process

Technologies

This project provided an excellent opportunity to deepen my expertise in React.js, focusing on component-based architecture to manage the quiz logic and user interactions dynamically. Bootstrap was used to create a responsive layout that adapts to different screen sizes, from smartphones to large desktop displays.

Usability Testing

Usability tests were conducted with a diverse group of participants to ensure the quiz interface was intuitive and met user expectations. This feedback was integral in refining the user interface to enhance the overall user experience.

Participants

  • High School Student (17 years old)
  • College Professor (45 years old)
  • Software Developer (30 years old)
  • Marketing Manager (38 years old)

Participants were asked to complete a set of quizzes, navigate through different categories, and provide feedback on the clarity of the questions and the interface. Adjustments were made based on their inputs to improve the usability and accessibility of the app.

Weather App

Description:
The Weather App is a comprehensive web application designed to provide real-time weather information for any location worldwide. This project leverages React.js for managing stateful components and Bootstrap for ensuring the app is accessible on any device, from mobile to desktop.

My Design Process

The focus was on creating an intuitive user interface that simplifies the complexity of weather data. The design prioritizes readability and ease of use, ensuring that users can quickly obtain the weather information they need without any hassle.

Wireframe

Color Palette

The color palette was chosen to visually represent various weather conditions effectively, enhancing user experience by using color coding to indicate temperature ranges, weather conditions, and alerts:

#085174 (Deep Ocean Blue)

#FFFFFF (Pure White)

#333 (Dark Gray)

#E0F7FA (Ice Blue)

#4CAF50 (Vibrant Green)

#2196F3 (Brilliant Blue)

My Coding Process

Technologies

This project was a perfect opportunity to refine my React.js skills, particularly for managing asynchronous API calls to OpenWeather and updating the UI state based on user interactions. Bootstrap's responsive features were integral in making the app functional across various devices.

Usability Testing

Usability testing was conducted to ensure the app met user expectations for accuracy and speed. Real-time feedback helped refine the UX to better serve user needs.

Participants

  • Weather Enthusiast (29 years old)
  • Professional Meteorologist (40 years old)
  • Outdoor Event Planner (35 years old)
  • High School Teacher (47 years old)
  • College Student (21 years old)
  • Retired Senior (65 years old)

Participants provided feedback on the app's ability to quickly deliver the needed forecasts and any additional features they desired.
